# 전략적 기획 워크플로우 (v1)

## Requirements Summary
- 목표: 아이디어를 실행 가능한 제품/기능 계획으로 전환한다.
- 범위: 문제정의, 우선순위, 실행계획, 리스크, 검증 기준까지 포함한다.
- 운영 원칙: 작은 실험으로 빠르게 검증하고, 근거 기반으로 의사결정한다.

## Acceptance Criteria
- 문제정의 문서에 `대상 사용자`, `핵심 문제`, `성공 지표`가 모두 명시된다.
- 옵션 비교표에 최소 2개 대안과 비용/효과/리스크가 정량 혹은 준정량으로 기록된다.
- 실행계획에 `담당자`, `마감일`, `완료조건(DoD)`이 각 작업마다 존재한다.
- 리스크 목록의 각 항목에 대응 전략(회피/완화/수용)과 트리거가 정의된다.
- 킥오프 전 체크리스트(요구사항, 일정, 의존성, 측정지표) 100% 충족.

## Implementation Steps
1. 문제 프레이밍
- 입력: 아이디어 1문장, 현재 상황, 제약(예산/일정/인력).
- 산출물: Problem Statement 1페이지, 성공 지표 3개 이내.

2. 목표/지표 정의 (North Star + Guardrail)
- 핵심 지표 1개, 보호 지표 2개 설정.
- 기준선(Baseline)과 목표치(Target), 측정 주기 확정.

3. 옵션 설계 및 전략 선택
- 옵션 A/B(필요 시 C) 설계.
- 점수표(영향도, 복잡도, 리스크, 리드타임)로 비교 후 1안 선택.
- ADR 기록: 왜 선택했고 무엇을 포기했는지 명시.

4. 실행 로드맵 분해
- 90일 기준으로 `Now(0-30일) / Next(31-60일) / Later(61-90일)` 분리.
- 작업 단위를 1~3일 크기로 쪼개고 담당자 할당.

5. 검증 실험 설계
- 각 가설에 대해 실험 1개 이상 매핑.
- 실험 템플릿: 가설, 방법, 성공/실패 기준, 중단 조건.

6. 리스크/의존성 관리
- 리스크 등록부 생성: 확률, 영향, 조기신호, 대응전략.
- 외부 의존성(API, 인프라, 타팀 승인) 일정 락인.

7. 실행 준비(Go/No-Go)
- 체크리스트 기반 승인 회의.
- 부족 항목은 보완 액션과 재검토 날짜 지정.

8. 운영 루프
- 주간: 진행률/블로커/지표 점검.
- 격주: 우선순위 재정렬.
- 월간: 전략 적합성 재평가(유지/확장/중단).

## Risks and Mitigations
- 문제정의 불명확: 초기에 사용자 인터뷰 3건 이상으로 가설 보정.
- 과도한 범위 확장: 변경요청은 영향도 평가 후 배치 릴리즈 반영.
- 지표 왜곡: 핵심지표와 보호지표를 동시에 모니터링.
- 실행 지연: 작업 크기 상한(3일) 유지, 병목 발견 시 즉시 재배분.

## Verification Steps
1. 문서 검증
- Problem Statement, 옵션 비교표, ADR, 로드맵, 리스크 등록부 존재 확인.

2. 수치 검증
- 각 목표 지표에 baseline/target/date가 모두 있는지 확인.

3. 실행성 검증
- 모든 작업에 담당자/기한/완료조건(DoD) 존재 여부 확인.

4. 실험성 검증
- 각 핵심 가설에 대응 실험과 중단 조건이 있는지 확인.

5. 운영성 검증
- 주간/격주/월간 리뷰 캘린더 이벤트 생성 확인.

## ADR
- Decision: 90일 시계열 기반의 실험-검증 중심 기획 워크플로우를 채택한다.
- Drivers: 불확실성 대응, 빠른 학습, 실행 관리 용이성.
- Alternatives considered:
  - 전통형 대규모 선계획(상세 명세 선작성)
  - 애드혹 실행(문서 최소화, 빠른 구현 우선)
- Why chosen: 선계획 대비 민첩하며, 애드혹 대비 재현성과 정렬(Alignment)이 높다.
- Consequences: 리뷰/문서화 오버헤드는 증가하지만 실패 비용과 재작업은 감소한다.
- Follow-ups:
  - 첫 2주 파일럿 후 지표/회의체 빈도 최적화
  - 팀 규모에 맞춰 역할(RACI) 상세화
